# 000. 프로젝트 킥오프: 사무직원이 바이브 코더 살아남기

> **결론**: 이 프로젝트는 "스토리가 곧 마케팅"이다. 직장 내 인정 → 제품 출시 → 수익화 순서로 실행하되, 그 과정 전체를 컨텐츠화한다. 지금 당장 결정해야 할 것은 3가지: ①컨텐츠 플랫폼, ②직장 내 첫 타깃, ③첫 제품 방향.

---

## 1. 이 프로젝트가 왜 이길 수 있는가

대부분의 개발 컨텐츠는 이미 잘하는 사람이 만든다. 이 프로젝트의 차별점:

| 일반 개발 유튜버 | 이 프로젝트 |
|---|---|
| IT 회사 재직 or 전공자 | 공기업 행정 사무직 |
| 코딩 → 컨텐츠 | 컨텐츠 = 코딩 과정 자체 |
| "이렇게 하면 됩니다" | "이렇게 하다 막혔습니다" |
| 결과물 중심 | 서사 중심 |

**이 서사를 원하는 타겟**: 코딩을 배우고 싶지만 진입 못 하는 직장인, 바이브 코딩에 관심 있는 비개발자, AI로 뭔가 만들어보고 싶은 사람들.

---

## 2. 전체 로드맵 초안

### Phase 1: 직장 내 인정 (0~6개월)
- 현 직장의 반복 업무 중 자동화/툴화 가능한 것 발굴
- 엑셀 매크로 → 웹 툴 → 내부 시스템 수준으로 단계 상승
- 목표: "저 사람 코드로 뭔가 만드네" 인식 확보

### Phase 2: 첫 제품 출시 (6~12개월)
- 직장 경험에서 나온 실제 문제를 외부 제품으로 전환
- 사전 홍보 = 제작 과정 스토리 (SNS/유튜브)
- 목표: 유료 사용자 첫 10명

### Phase 3: 수익화 & 스케일업 (12개월~)
- 제품 수익이 월 50만원 이상 → 다음 스텝 논의
- 컨텐츠 자체 수익 (광고, 스폰서, 강의) 추가

---

## 3. 지금 당장 결정해야 할 3가지

### 결정 1: 컨텐츠 플랫폼
선택지와 득실:

| 플랫폼 | 장점 | 단점 | 추천 시작 시기 |
|---|---|---|---|
| 유튜브 | 알고리즘 유입, 구독자 자산화 | 편집 비용, 초기 진입 느림 | 3개월 후 |
| 블로그 (티스토리/노션) | 글쓰기 빠름, SEO | 느린 성장 | 지금 당장 |
| X(트위터)/스레드 | 빠른 피드백, 네트워크 | 휘발성 | 지금 당장 |
| 유튜브 쇼츠 | 빠른 알고리즘 | 짧은 포맷 한계 | 1개월 후 |

**권장 순서**: 글(블로그/X) → 쇼츠 → 유튜브 롱폼

### 결정 2: 직장 내 첫 타깃 업무
지금 직장에서 반복하는 업무 중 뭘 자동화하면 임팩트가 클까?

예시 카테고리:
- 보고서 취합/정리
- 데이터 입력/변환
- 문서 서식 작업
- 민원 처리 반복 업무
- 일정/회의 관련 행정

→ **다음 대화에서 구체화 필요**

### 결정 3: 첫 제품 방향
3가지 접근법:

**A. 직장 내부 도구 외부화**
- 만든 내부 툴을 SaaS로 포장
- 검증된 문제, 검증된 수요

**B. 바이브 코딩 학습 도구**
- 비개발자를 위한 뭔가
- 내가 겪는 문제를 제품으로

**C. 컨텐츠 연동 제품**
- 유튜브/블로그 구독자 대상
- 커뮤니티 기반 수익화

→ **아직 결정 불필요, 6개월 내 자연스럽게 결정됨**
